Class Layout

The user can configure which classes are displayed in the main window for immediate selection. The configuration file wspec/layout.wrm holds this information in simple text format. The user may decide which classes he/she most frequently accesses and arrange them into groups etc..

The "Class Layout Editor" is effectivly a graphical editor for this configuration. Click on the "Set Selection" button in the main window to start the editor.

In the yellow area the classes are displayed in the same layout as in the main class window. You can click on any class-name and drag it to a new position in the text grid.
Below the line are the remaining visible classes which can also be used for the default. Most often these left-over classes belong to the system, but they may be of interest to some users. The classes "KeySet" and "Model" are usually included in the class layout.
You can pick up one of these classes and drag it into the yellow area to position it in the layout.
To remove a class from the layout, just drag its name below the line into the area of the remaining classes.

Menu items:

  • Quit: quits the window, if you have write access to the wspec directory and you have made any unsaved modification you'll be asked whether you want to save them. Answer "No" to cancel the edits.
  • Help: brings up this help.
  • Print: lets you print a snapshot of the window.
  • Apply: copies the current layout to the main window. The new layout will not be permanent until you "Save".
  • Save: saves the current layout to wspec/layout.wrm in the database directory. Note, this button will not appear if you don't have write access to this file.
